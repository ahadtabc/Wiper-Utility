Key Features

Targeted System Cleanup

Safely wipe files from essential, high-volume cleanup zones: **User Temp Files**, **System Cache (Temp)**, and **Prefetch** folders. Requires admin privileges for system-level paths.

Custom Directory Wiping

Specify any custom folder path on your drive for deep cleaning. The utility ensures that the folder itself remains intact while securely deleting all its contents.

Startup Management & Aesthetics

Toggle application launch at Windows startup using the **Registry**. Easily switch between **Dark/Light Mode** for comfortable use in any environment.

Built with Modern Python

WiperClean is a lean utility built entirely with **Python**. We chose the high-performance **CustomTkinter** framework to deliver a native, feature-rich graphical user interface (GUI) without the heavy resource footprint of web-based applications.

  • ✓ **Windows Registry Integration:** Manages startup keys using the native `winreg` library.
  • ✓ **Privilege Checking:** Uses `ctypes` to ensure critical system cleans are only attempted with **Administrator** rights.
  • ✓ **Safe Deletion Logic:** Utilizes `os` and `shutil` for content removal while maintaining folder integrity, preventing system errors.

How The Cleanup Logic Works

The cleanup process is structured to prioritize safety and permissions:

  1. Step 1: Privilege Check

    The utility first determines if it is running with Administrator privileges. This is crucial as system-wide paths like Prefetch cannot be accessed otherwise.

  2. Step 2: System Path Iteration

    It loops through enabled system items (Temp, Cache, Prefetch). If an item requires admin and the check fails, the item is safely **skipped** and logged with a warning.

  3. Step 3: Safe Directory Cleaning

    The `_clean_directory` function is called. This function iterates through every file and folder *inside* the target path and deletes them individually, ensuring the root folder itself is never deleted, which prevents system instability.

  4. Step 4: Custom Path Execution

    If a Custom Path is entered, the **same safe cleaning logic** is applied to its contents for complete data removal from user-specified locations.
